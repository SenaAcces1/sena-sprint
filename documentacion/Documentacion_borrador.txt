Resumen Técnico: SenaAccess

"SenaAccess" es un sistema integral de gestión de accesos y reportes diseñado para optimizar el control de entradas y salidas en ambientes institucionales. El proyecto destaca por un enfoque de desarrollo moderno que prioriza la seguridad, la interactividad y la eficiencia administrativa.

Arquitectura: SPA con Backend Laravel

El sistema implementa una arquitectura desacoplada dentro de un entorno Laravel:

Estructura: Utiliza Laravel como API robusta para la gestión de datos y seguridad, mientras que la interfaz de usuario es una Single Page Application (SPA) construida íntegramente en React.

Ventaja principal: Proporciona una experiencia de usuario fluida sin recargas de página, delegando la lógica de negocio al servidor y la interactividad al cliente mediante Axios y React Router.

Herramientas de Comunicación y Navegación

1. Axios (Cliente HTTP)
¿Qué es?: Es una librería basada en promesas para realizar solicitudes HTTP desde el navegador hacia el servidor.
Uso en el proyecto: 
- Se utiliza para enviar y recibir datos en formato JSON desde los controladores de Laravel.
- Posee interceptores globales (configurados en bootstrap.js) que adjuntan automáticamente el Token de Acceso (Bearer Token) en el encabezado de cada petición si el usuario está autenticado.
- Facilita el manejo de errores del servidor y la comunicación asíncrona en formularios de Login, Registro y el Panel Administrativo.

2. React Router (Enrutamiento Dinámico)
¿Qué es?: Es la librería estándar para gestionar la navegación en aplicaciones de una sola página (SPA).
Uso en el proyecto:
- Permite que la URL del navegador cambie y se carguen diferentes componentes (Login, Admin, Fingerprint) sin necesidad de recargar toda la página.
- Utiliza el hook useNavigate para redireccionar a los usuarios después de acciones exitosas (como iniciar sesión).
- Gestiona la estructura de rutas en el archivo app.jsx, asegurando que cada vista sea modular y accesible.

Flujo de Trabajo (Workflow) General

1. Autenticación: El usuario ingresa sus credenciales en el componente Login.
2. Generación de Token: El servidor (Laravel Sanctum) valida los datos y retorna un JSON con un Access Token de corta duración.
3. Persistencia Local: React captura el token y lo almacena en el LocalStorage del navegador, junto con el rol del usuario (Admin/Usuario).
4. Navegación Protegida: React Router redirige al usuario al panel correspondiente según su rol.
5. Peticiones Autorizadas: Al navegar por el sistema (ej. cargar la lista de usuarios en Admin), Axios lee el token del LocalStorage y lo envía en el encabezado Authorization de forma automática.
6. Gestión de Datos: El administrador puede realizar operaciones CRUD (Crear, Leer, Actualizar, Eliminar) que se sincronizan en tiempo real con la base de datos MySQL mediante controladores API.

------------------------

Flujo de Trabajo para Creación de Vistas y Componentes

Para mantener la consistencia visual y funcional del proyecto, se sigue este proceso al añadir nuevas interfaces:

1. Creación del Componente (JSX):
   - Se crea un nuevo archivo en `resources/js/components/` (ej. `Novedades.jsx`).
   - Se utilizan componentes funcionales de React con Hooks (`useState`, `useEffect`) para manejar el estado local y los ciclos de vida.

2. Aplicación de Estilos y Temas:
   - Se utilizan las clases globales definidas en `app.css` como `glass-box` (para el efecto de transparencia), `fade-in-up` (para animaciones de entrada) y `btn-glow` (para botones estilo neón).
   - Se aprovechan las variables CSS (`--primary-color`, `--glass-bg`) para asegurar que el componente sea compatible con el modo claro y oscuro.

3. Configuración de la Ruta:
   - En `resources/js/app.jsx`, se importa el nuevo componente.
   - Se añade una etiqueta `<Route path="/nombre-ruta" element={<NuevoComponente />} />` dentro del componente `<Routes>`.

4. Conexión con Datos (Axios):
   - Si la vista requiere datos del servidor, se utiliza `axios.get()` o `axios.post()` dentro de un `useEffect` o manejador de eventos.
   - Gracias a la configuración global, no es necesario añadir manualmente el token de seguridad en cada nuevo componente.

5. Previsualización y Compilación:
   - Se ejecuta `npm run dev` para que Vite procese los cambios instantáneamente (HMR - Hot Module Replacement) sin perder el estado de la aplicación.

Funcionalidades Clave
1. Gestión de Identidad: Registro y autenticación de usuarios con roles definidos.
2. Control de Flujo: Registro en tiempo real de ingresos y egresos vinculados a biometría.
3. Seguridad Robusta: Protección de rutas tanto en frontend (React Router) como en backend (Laravel Sanctum).
4. Interfaz Adaptativa: Soporte para modo oscuro/claro y diseño optimizado para diferentes dispositivos.
5. Reportes Administrativos: Visualización de datos y estadísticas de acceso.
